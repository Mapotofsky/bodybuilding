# P0 详细设计文档 — 核心训练

> **版本**：v1.0 &ensp;|&ensp; **日期**：2026-03-28 &ensp;|&ensp; **状态**：已完成
>
> **范围**：M1 用户与鉴权 + M2 训练日志（记录/补录/编辑/复制/分享/删除）+ 动作库基础 CRUD

---

## 1 模块概述

P0 阶段交付 IronLog 的最小可用产品，包含：

- 用户注册/登录/令牌刷新
- 个人资料查看与编辑
- 训练日志的完整 CRUD（创建、查询、编辑、删除）
- 训练复制与分享数据生成
- 内置动作库（50+ 预置动作）及自定义动作
- 动作历史记录查询
- 训练计时器（前端）

## 3 数据库设计

### 3.1 ER 关系

```
User (1) ──→ (N) Workout (1) ──→ (N) WorkoutExercise (1) ──→ (N) WorkoutSet
User (1) ──→ (N) Exercise (自定义动作, is_custom=true)
Exercise (1) ←── (N) WorkoutExercise
```

## 4 后端 API 设计

基础路径：`/api/v1`，认证方式：Bearer Token (JWT)。

### 4.1 认证模块 (`/auth`)

#### POST /auth/register

- **请求**：`{ email: EmailStr, password: str (≥6), nickname?: str }`
- **响应 201**：`{ access_token, refresh_token, token_type: "bearer" }`
- **错误 409**：邮箱已注册
- **逻辑**：bcrypt 散列密码，创建用户，返回双令牌

#### POST /auth/login

- **请求**：`{ email: EmailStr, password: str }`
- **响应 200**：`{ access_token, refresh_token, token_type: "bearer" }`
- **错误 401**：邮箱或密码错误
- **逻辑**：校验邮箱存在性 + bcrypt 比对

#### POST /auth/refresh

- **请求**：`{ refresh_token: str }`
- **响应 200**：`{ access_token, refresh_token, token_type: "bearer" }`
- **错误 401**：刷新令牌无效
- **逻辑**：解码 refresh token，验证 type=refresh，生成新双令牌

### 4.2 用户模块 (`/users`)

#### GET /users/me

- **认证**：必须
- **响应 200**：`UserOut { id, email, nickname, avatar_url, gender, height, weight, birth_date, role, created_at }`

#### PUT /users/me

- **认证**：必须
- **请求**：`UserUpdate { nickname?, avatar_url?, gender?, height?, weight?, birth_date? }`（仅更新传入的字段）
- **响应 200**：`UserOut`

### 4.3 动作库模块 (`/exercises`)

#### GET /exercises

- **认证**：必须
- **查询参数**：`category?: str`, `q?: str`（名称模糊搜索）
- **响应 200**：`ExerciseOut[]`
- **逻辑**：返回所有系统预置动作 + 当前用户的自定义动作，按 category + name 排序

#### POST /exercises

- **认证**：必须
- **请求**：`{ name, category, type?, description?, met_value? }`
- **响应 201**：`ExerciseOut`
- **逻辑**：创建自定义动作，`is_custom=true`，`user_id=当前用户`

#### GET /exercises/{exercise_id}/history

- **认证**：必须
- **查询参数**：`limit?: int (default=30, max=100)`
- **响应 200**：`{ date, set_number, weight, reps, unit, rest_seconds }[]`
- **逻辑**：联表查询 Workout → WorkoutExercise → WorkoutSet，筛选当前用户 + 指定动作，按日期倒序

### 4.4 训练日志模块 (`/workouts`)

#### POST /workouts

- **认证**：必须
- **请求**：
```json
{
  "date": "2026-03-28",
  "start_time": "2026-03-28T10:00:00+08:00",
  "end_time": null,
  "note": "腿部训练",
  "mood": 4,
  "plan_template_id": null,
  "exercises": [
    {
      "exercise_id": 30,
      "sort_order": 0,
      "superset_group": null,
      "sets": [
        { "set_number": 1, "weight": 100, "reps": 8, "unit": "kg", "rpe": 7.5, "is_warmup": false, "is_dropset": false, "is_failure": false }
      ]
    }
  ]
}
```
- **响应 201**：`WorkoutOut`（完整嵌套结构）
- **逻辑**：创建 Workout → 遍历创建 WorkoutExercise → 遍历创建 WorkoutSet → 返回完整对象

#### GET /workouts

- **认证**：必须
- **查询参数**：`month?: "YYYY-MM"`, `from?: date`, `to?: date`
- **响应 200**：`WorkoutSummary[]`
- **WorkoutSummary 结构**：
```json
{
  "id": 1,
  "date": "2026-03-28",
  "start_time": "...",
  "end_time": "...",
  "note": "...",
  "mood": 4,
  "exercise_count": 5,
  "total_sets": 20,
  "total_volume": 12500.0,
  "plan_template_id": null,
  "template_name": null,
  "template_color": null,
  "plan_color": null,
  "exercise_ids": [30],
  "created_at": "..."
}
```
- **逻辑**：按 month 或 from/to 筛选，selectinload 关联数据，计算 exercise_count / total_sets / total_volume，按日期倒序

#### GET /workouts/{workout_id}

- **认证**：必须
- **响应 200**：`WorkoutOut`（含 exercises[].exercise_name, exercises[].exercise_category）
- **错误 404**：训练记录不存在

#### PUT /workouts/{workout_id}

- **认证**：必须
- **请求**：`WorkoutUpdate`（所有字段可选；若传入 exercises，执行全量替换）
- **逻辑**：更新基本字段 → 若 exercises 不为 null，删除旧 WorkoutExercise（级联删除 Sets），重新创建
- **响应 200**：`WorkoutOut`

#### DELETE /workouts/{workout_id}

- **认证**：必须
- **响应 204**：无内容
- **逻辑**：级联删除 Workout 及其下所有关联数据

#### POST /workouts/{workout_id}/copy

- **认证**：必须
- **请求**：`{ target_date: "2026-04-01" }`
- **响应 201**：`WorkoutOut`（新训练记录）
- **逻辑**：读取源训练 → 创建新 Workout（target_date）→ 复制所有 exercises 和 sets → 返回新记录

#### POST /workouts/{workout_id}/share

- **认证**：必须
- **响应 200**：`WorkoutShareData`
```json
{
  "date": "2026-03-28",
  "mood": 4,
  "duration_minutes": 65,
  "exercise_count": 5,
  "total_sets": 20,
  "total_volume": 12500.0,
  "exercises": [
    { "name": "杠铃深蹲", "category": "legs", "sets": 4, "volume": 3200.0 }
  ],
  "note": "腿部训练"
}
```
- **逻辑**：计算每个动作的组数和容量，计算训练时长

### 5.5 页面详细设计

#### 5.5.1 HomePage

- **数据**：
  - `getWorkouts({ month: 当月 })` 取本月训练列表
  - `getCalendar(今天, 今天)` 取今日计划条目（P1 新增）
- **展示**：
  - 欢迎语（用户昵称 + 今日日期）
  - "开始训练" 按钮 → `/workouts/new`（进入计划/模版选择 + 全动作库流程）
  - **今日计划区域**（P1 新增，有安排时显示）：
    - 今日模版卡片：模版名 + 计划名 + 颜色标识
    - 「开始」→ `/workouts/new?template_id=X`（自动激活模版过滤）
    - 已完成条目显示已完成状态标签
  - 统计卡片：本月训练次数、本月总容量
  - 最近训练列表（日期、心情 emoji、动作数/组数/容量、模版名标签）

#### 5.5.2 WorkoutsPage

- **数据**：按月查询训练列表
- **交互**：
  - 月份选择器（左右箭头切换）
  - 月度统计（训练次数/总组数/总容量）
  - 训练卡片列表（点击进入详情）
  - 右下角 FAB "+" 按钮 → `/workouts/new`

#### 5.5.3 WorkoutCreatePage

多阶段训练记录流程，共四个 Phase：

| Phase | 触发条件 | 说明 |
|-------|----------|------|
| `select` | 初始 / 每组完成后返回 | 动作选择。首次进入时设置日期、单位、可选计划/模版；后续进入显示本次已记录组的摘要 |
| `training` | 选中动作后进入 | 单组录入：动作名 + 组序号 + 模版备注（若激活）；输入重量/次数；提交后跳到 rest |
| `rest` | 每组提交后自动进入 | 组间休息计时；显示本组完成数据和模版备注；可跳过或继续下一组/换动作 |
| `finish` | 手动点击"结束训练" | 总结：选择心情（1-5）、填写备注、确认保存；调用 `createWorkout` 后跳转详情页 |

**模版激活行为**（`?template_id=X` URL 参数 或 页内计划/模版选择器触发）：
- 动作选择列表**仅显示**模版内动作；**非模版动作不出现**在列表中（过滤，非排序/置顶）
- 用户可点击「清除选择」回到全动作库
- training / rest 阶段在动作信息区展示模版 `note` 字段内容
- 保存时 payload 携带 `plan_template_id`，后端写入 workouts 表，`WorkoutSummary` 返回 `template_name`

#### 5.5.4 WorkoutDetailPage

- **功能**：
  - 展示完整训练信息（日期、时长、心情、备注）
  - 各动作及其组数详情（重量 × 次数，标记 badge）
  - 操作按钮：编辑 / 复制 / 分享 / 删除
  - 删除需 ConfirmDialog 确认

#### 5.5.5 WorkoutEditPage

- 与 CreatePage 类似，预填现有数据
- 提交时调用 `updateWorkout` 全量替换 exercises

#### 5.5.6 ProfilePage

- 展示/编辑用户资料（昵称、性别、身高、体重、生日）
- 退出登录按钮

## 6 预置动作数据

| 分类 | 数量 | 示例 |
|------|------|------|
| 胸部 (chest) | 10 | 平板杠铃卧推、上斜哑铃卧推、蝴蝶机夹胸 |
| 背部 (back) | 8 | 杠铃划船、引体向上、硬拉 |
| 腿部 (legs) | 9 | 杠铃深蹲、罗马尼亚硬拉、腿举 |
| 肩部 (shoulders) | 7 | 杠铃推举、哑铃侧平举、面拉 |
| 手臂 (arms) | 8 | 杠铃弯举、绳索下压、过头臂屈伸 |
| 核心 (core) | 5 | 卷腹、平板支撑、悬垂举腿 |
| 有氧 (cardio) | 5 | 跑步机、划船机、跳绳 |

每个动作均配置了 MET 值，供后续 P3 热量估算使用。

## 7 安全设计

| 项 | 实现 |
|---|------|
| 密码存储 | bcrypt 散列（自动 salt） |
| 令牌签发 | JWT HS256，access_token 15min，refresh_token 30d |
| 令牌验证 | HTTPBearer 中间件，解码后查库确认用户存在 |
| 自动刷新 | 前端 axios 拦截器，401 时自动用 refresh_token 换取新双令牌 |
| 数据隔离 | 所有 workout/exercise 查询均 where user_id = 当前用户 |
| CORS | 仅允许 localhost:5173 |
