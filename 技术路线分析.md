## 2 技术选型决策

### 2.1 后端：Python 3.11 + FastAPI

本机 Python + Anaconda 就绪，PDF/OCR 库（PyMuPDF、PaddleOCR）均为 Python 生态，无需跨语言调用。FastAPI 自动生成 API 文档（Swagger UI），Pydantic 校验与数据模型天然契合。记录型应用无高并发需求，单进程 Uvicorn 足够。

### 2.2 前端：React + TypeScript + Vite（PWA）

构建为响应式 Web 应用，手机浏览器可添加到桌面，等效原生 App 体验。相比 React Native / Flutter，Web 应用开发调试更简单，无需处理原生构建链。图表（Recharts）、日历、Markdown 渲染等 Web 生态组件成熟丰富。

仅需安装 Node.js： `winget install OpenJS.NodeJS.LTS`

### 2.3 数据库：PostgreSQL 16（本机已运行）

单库即可满足所有需求。JSONB 存储体检指标等半结构化数据，GIN 索引支持 Wiki 全文搜索。无需引入 Redis。

### 2.4 文件存储：本地文件系统

上传文件直接存储到 `./uploads/` 目录，无需 MinIO 或对象存储服务。

### 2.5 PDF 解析与 OCR

| 环节 | 工具 |
|------|------|
| 文本型 PDF | **PyMuPDF (fitz)** |
| 扫描型 PDF | **PaddleOCR**（本机有 pytorch 环境可加速） |
| 指标提取 | 正则匹配 |

### 2.6 后台任务

使用 FastAPI 内置的 **BackgroundTasks**，无需 Celery 或消息队列。PDF 解析在后台线程完成，用户上传后立即返回。

### 2.7 前端技术细节

| 层面 | 技术 |
|------|------|
| 框架 | React 18 + TypeScript + **Vite** |
| 路由 | React Router v6 |
| 状态管理 | Zustand + TanStack Query |
| UI | **TailwindCSS** + **shadcn/ui** |
| 图表 | **Recharts**（RM 曲线、容量统计、热量消耗） |
| 日历 | **react-big-calendar** 或自定义日历组件 |
| Markdown | **react-markdown**（Wiki 正文渲染） |
| 网络请求 | axios + TanStack Query |
| PWA | vite-plugin-pwa（离线缓存、桌面安装） |

## 5 环境准备清单

仅需两步即可开始开发，无需 Docker、无需额外服务。

### 5.1 安装 Node.js

```bash
winget install OpenJS.NodeJS.LTS
# 验证：node -v && npm -v
```

### 5.2 后端环境

本机python后端开发均使用appdev环境

```bash
conda create -n appdev python=3.11 -y
conda activate appdev
pip install fastapi "uvicorn[standard]" "sqlalchemy[asyncio]" asyncpg alembic
pip install pyjwt bcrypt python-multipart
pip install pymupdf paddleocr pandas numpy
```

### 5.3 前端环境

```bash
npm create vite@latest ironlog-frontend -- --template react-ts
cd ironlog-frontend
npm install
npm install @tanstack/react-query zustand axios recharts
npm install -D tailwindcss @tailwindcss/vite
```

### 5.4 数据库

PostgreSQL 使用本机已运行的实例，只需创建数据库：

```sql
CREATE DATABASE ironlog;
CREATE USER ironlog WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE ironlog TO ironlog;
```

## 7 技术风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| PaddleOCR 模型体积大（~150MB） | 使用 PP-OCRv4 轻量模型；按需延迟加载 |
| PostgreSQL 中文全文搜索精度 | 安装 `zhparser` 扩展提升分词质量 |
| PWA 功能受限（无原生推送等） | 核心功能均为 CRUD，不依赖原生能力；后期可用 Capacitor 包装为原生 App |
