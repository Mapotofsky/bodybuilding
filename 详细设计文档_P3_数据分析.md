## 1 模块概述

P3 阶段新增三大数据分析功能：

1. **RM 计算器**：支持多种经典公式（Epley/Brzycki/Lombardi/O'Conner），手动计算 + 从训练日志自动提取 e1RM 趋势。
2. **训练容量统计**：按动作/肌群/周/月维度聚合训练容量，生成折线图、柱状图、饼图。
3. **热量消耗估计**：基于 MET 值 × 体重 × 时长估算训练热量消耗。

## 2 数据库设计

P3 不新增数据表，所有分析数据均从现有表实时聚合计算：

- `workouts` + `workout_exercises` + `workout_sets` — 训练容量、RM 数据源
- `exercises` — MET 值、分类（用于肌群映射）
- `users` — 体重（用于热量计算）

### 2.1 新增视图（可选优化）

当数据量增大后，可创建物化视图加速常用查询：

```sql
CREATE MATERIALIZED VIEW mv_daily_volume AS
SELECT
    w.user_id,
    w.date,
    we.exercise_id,
    e.category,
    COUNT(ws.id) AS set_count,
    SUM(COALESCE(ws.weight, 0) * COALESCE(ws.reps, 0)) AS volume,
    MAX(ws.weight) AS max_weight,
    MAX(ws.reps) FILTER (WHERE ws.weight = (SELECT MAX(weight) FROM workout_sets ws2 WHERE ws2.workout_exercise_id = we.id)) AS reps_at_max_weight
FROM workouts w
JOIN workout_exercises we ON we.workout_id = w.id
JOIN workout_sets ws ON ws.workout_exercise_id = we.id
JOIN exercises e ON e.id = we.exercise_id
WHERE ws.is_warmup = false
GROUP BY w.user_id, w.date, we.exercise_id, e.category;
```

初期直接查询原始表，性能不足时再引入。

## 3 后端 API 设计

### 3.1 RM 计算器 (`/analysis/rm-calc`)

#### POST /api/v1/analysis/rm-calc

手动 RM 计算。

**请求**：

```json
{
  "weight": 100,
  "reps": 5,
  "formula": "epley"
}
```

**响应 200**：

```json
{
  "estimated_1rm": 116.7,
  "formula": "epley",
  "rm_table": [
    { "reps": 1, "weight": 116.7 },
    { "reps": 2, "weight": 113.1 },
    { "reps": 3, "weight": 109.9 },
    { "reps": 5, "weight": 100.0 },
    { "reps": 8, "weight": 90.5 },
    { "reps": 10, "weight": 84.4 },
    { "reps": 12, "weight": 79.4 },
    { "reps": 15, "weight": 73.0 },
    { "reps": 20, "weight": 64.8 }
  ]
}
```

**公式实现**：

```python
def calc_1rm(weight: float, reps: int, formula: str = "epley") -> float:
    if reps <= 0:
        raise ValueError("reps must be > 0")
    if reps == 1:
        return weight
    match formula:
        case "epley":
            return weight * (1 + reps / 30)
        case "brzycki":
            return weight * 36 / (37 - reps)
        case "lombardi":
            return weight * (reps ** 0.10)
        case "oconner":
            return weight * (1 + 0.025 * reps)
        case _:
            raise ValueError(f"Unknown formula: {formula}")

def calc_rm_table(one_rm: float, formula: str = "epley") -> list[dict]:
    """Given 1RM, calculate weight for various rep ranges."""
    targets = [1, 2, 3, 5, 8, 10, 12, 15, 20]
    result = []
    for r in targets:
        if r == 1:
            result.append({"reps": 1, "weight": round(one_rm, 1)})
        else:
            match formula:
                case "epley":
                    w = one_rm / (1 + r / 30)
                case "brzycki":
                    w = one_rm * (37 - r) / 36
                case "lombardi":
                    w = one_rm / (r ** 0.10)
                case "oconner":
                    w = one_rm / (1 + 0.025 * r)
            result.append({"reps": r, "weight": round(w, 1)})
    return result
```

### 3.2 RM 趋势 (`/analysis/rm-history`)

#### GET /api/v1/analysis/rm-history

从训练日志自动提取每个动作的 e1RM 趋势。

**查询参数**：

- `exercise_id: int`（必填）
- `from?: date`
- `to?: date`
- `formula?: str` (default="epley")

**响应 200**：

```json
{
  "exercise_id": 1,
  "exercise_name": "平板杠铃卧推",
  "formula": "epley",
  "data_points": [
    { "date": "2026-01-15", "estimated_1rm": 95.0, "best_set": { "weight": 80, "reps": 6 } },
    { "date": "2026-02-01", "estimated_1rm": 100.0, "best_set": { "weight": 85, "reps": 5 } },
    { "date": "2026-03-10", "estimated_1rm": 106.7, "best_set": { "weight": 90, "reps": 5 } }
  ],
  "current_estimated_1rm": 106.7,
  "progress_pct": 12.3
}
```

**逻辑**：

1. 查询该动作所有非热身组的 (date, weight, reps)
2. 按日期分组，每天取 e1RM 最高的组（best_set）
3. 计算每天的 estimated_1rm
4. progress_pct = (最新 - 最早) / 最早 × 100%

**SQL 核心**：

```sql
SELECT w.date, ws.weight, ws.reps
FROM workouts w
JOIN workout_exercises we ON we.workout_id = w.id
JOIN workout_sets ws ON ws.workout_exercise_id = we.id
WHERE w.user_id = :user_id
  AND we.exercise_id = :exercise_id
  AND ws.is_warmup = false
  AND ws.weight IS NOT NULL
  AND ws.reps IS NOT NULL
  AND ws.reps > 0
ORDER BY w.date
```

### 3.4 热量消耗统计 (`/analysis/calories`)

#### GET /api/v1/analysis/calories

**查询参数**：`from?: date`, `to?: date`

**响应 200**：

```json
{
  "period": { "from": "2026-03-01", "to": "2026-03-28" },
  "total_calories": 4520,
  "user_weight_kg": 75,
  "daily_data": [
    {
      "date": "2026-03-25",
      "calories": 380,
      "duration_minutes": 65,
      "exercises": [
        { "name": "杠铃深蹲", "met_value": 6.0, "duration_min": 25, "calories": 188 },
        { "name": "腿举", "met_value": 5.0, "duration_min": 15, "calories": 94 }
      ]
    }
  ]
}
```

**计算模型**：

```python
def estimate_calories(met: float, weight_kg: float, duration_hours: float) -> float:
    """热量(kcal) = MET × 体重(kg) × 时长(h)"""
    return met * weight_kg * duration_hours
```

**每动作时长估算**：

> **前置说明**：P1 已实现的 `Workout` 模型包含 `start_time` 和 `end_time` 字段（用户在 WorkoutCreatePage 手动输入，非计时器自动记录）。该字段可能为空（用户未填写）。

- 若训练有 `start_time` 和 `end_time`，总时长 = `end_time - start_time`
- 每动作时长按比例分配：动作组数 / 总组数 × 总时长
- 若 `start_time` 或 `end_time` 为空，按经验值估算：每组约 3 分钟（含组间休息）

## 6 业务规则

### 6.1 e1RM 提取规则

1. 仅取 `is_warmup = false` 的组
2. 仅取 weight > 0 且 reps > 0 的组
3. 同一训练日的同一动作，取 e1RM 最高的组作为当日代表
4. reps > 30 时 RM 公式精度下降，限制 reps ≤ 30

### 6.2 肌群映射

exercise.category 直接映射为肌群（P0 已有分类），compound 类动作可配置主要/次要肌群（P2 Wiki 的 target_muscles 可复用）。

### 6.3 热量估算精度说明

前端应明确标注"估算值"，热量计算受体重精度、实际训练强度、休息时间等因素影响，仅供参考。

### 6.4 首页 HomePage 更新

在 P0 首页基础上新增：

- "本月摘要" 卡片：调用 `/analysis/summary?period=month`，展示训练次数、总容量、总热量
- "RM 进步" 卡片：展示最近有进步的 TOP 3 动作
