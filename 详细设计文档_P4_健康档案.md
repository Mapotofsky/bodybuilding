## 1 模块概述

P4 阶段构建个人健康档案系统：

1. **体测记录**：手动录入体重、体脂率、围度等时序数据，追踪身体变化。
2. **PDF 体检报告导入**：上传体检 PDF，PyMuPDF 提取文本，正则匹配提取关键指标，PaddleOCR 处理扫描型 PDF。
3. **健康目标**：设定体重/体脂/围度目标，追踪完成进度。
4. **健康档案总览**：整合所有健康数据，展示趋势图表。

## 4 PDF 解析服务

### 4.1 解析流程

```python
async def parse_medical_report(report_id: int):
    """BackgroundTask: 解析体检 PDF 并提取指标。"""
    # 1. 读取 MedicalReport 记录
    # 2. 更新 parse_status = "processing"
    # 3. 使用 PyMuPDF 提取文本
    text = extract_text_pymupdf(file_path)
    # 4. 若文本为空或过短，尝试 OCR
    if len(text.strip()) < 50:
        text = extract_text_ocr(file_path)
    # 5. 正则匹配提取指标
    indicators = extract_indicators(text)
    # 6. 保存结果
    report.indicators = indicators
    report.raw_text = text
    report.parse_status = "completed"
    # 异常处理：parse_status = "failed", parse_error = str(e)
```

### 4.2 文本提取

```python
import fitz  # PyMuPDF

def extract_text_pymupdf(file_path: str) -> str:
    doc = fitz.open(file_path)
    text = ""
    for page in doc:
        text += page.get_text()
    doc.close()
    return text
```

### 4.3 OCR 提取（扫描型 PDF）

```python
def extract_text_ocr(file_path: str) -> str:
    from paddleocr import PaddleOCR
    import fitz

    ocr = PaddleOCR(use_angle_cls=True, lang="ch")
    doc = fitz.open(file_path)
    full_text = ""
    for page in doc:
        pix = page.get_pixmap(dpi=300)
        img_bytes = pix.tobytes("png")
        result = ocr.ocr(img_bytes, cls=True)
        for line in result:
            for word_info in line:
                full_text += word_info[1][0] + " "
            full_text += "\n"
    doc.close()
    return full_text
```

PaddleOCR 按需延迟导入，避免启动时加载大模型。

```python
        r"谷草转氨酶[：:\s]*(\d+\.?\d*)",
        r"AST[：:\s]*(\d+\.?\d*)",
    ],
    "creatinine": [
        r"肌酐[：:\s]*(\d+\.?\d*)",
        r"Cr[：:\s]*(\d+\.?\d*)",
    ],
    "uric_acid": [

        r"UA[：:\s]*(\d+\.?\d*)",
    ],
    "hemoglobin": [
        r"血红蛋白[：:\s]*(\d+\.?\d*)",
        r"Hb[：:\s]*(\d+\.?\d*)",
        r"HGB[：:\s]*(\d+\.?\d*)",
    ],
}

def extract_indicators(text: str) -> dict:
    indicators = {}
    for key, patterns in INDICATOR_PATTERNS.items():
        for pattern in patterns:
            match = re.search(pattern, text, re.IGNORECASE)
            if match:
                try:
                    indicators[key] = float(match.group(1))
                except ValueError:
                    pass
                break
    return indicators
```

## 7 业务规则

### 7.1 体重与 BMI 联动

录入体重时，若用户 profile 有身高数据，自动计算并展示 BMI：

```
BMI = 体重(kg) / 身高(m)²
```

BMI 分类：
- < 18.5：偏瘦
- 18.5-24.0：正常
- 24.0-28.0：超重
- ≥ 28.0：肥胖

### 7.2 目标自动达成检测

每次录入体测数据后，检查活跃目标：

- 若 current_value 达到/超过 target_value（减重类目标方向相反），自动标记 status=achieved
- 前端弹出祝贺提示

### 7.3 文件存储路径

```
uploads/
└── medical/
    └── {user_id}/
        └── {uuid}.pdf
```

文件名使用 UUID 避免冲突，原始文件名存入数据库。

### 7.4 解析轮询机制

前端上传后跳转到详情页，若 parse_status 为 pending/processing，每 3 秒调用 `GET /profile/medical-reports/{id}` 检查状态，直到 completed 或 failed。

### 7.5 users 表体重同步

录入体测数据时，若包含 weight 字段，同步更新 `users.weight`，确保热量计算（P3）使用最新体重。

## 9 依赖安装说明

P4 所需的 PDF/OCR 依赖较重，建议按需安装：

```bash
# 基础 PDF 解析（必装，~15MB）
pip install pymupdf

# OCR 功能（可选，~150MB 模型文件）
pip install paddleocr paddlepaddle
```

PaddleOCR 仅在遇到扫描型 PDF 时才会被延迟导入加载。若用户不需要 OCR 功能，无需安装 paddleocr 和 paddlepaddle。
