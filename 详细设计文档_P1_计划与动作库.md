## 1 模块概述

P1 阶段新增两大功能块：

1. **训练计划模块**：用户可创建训练计划，安排练休分布，支持按周循环、按日循环、机动三种编排模式。训练时可选择计划/模版，动作选择列表**仅显示**模版内动作（过滤，非排序）。
2. **动作库浏览增强**：在 P0 基础列表之上，新增动作详情页，展示动作描述、目标肌群、历史使用统计。

  "mode": "cyclic",
  "cycle_length": 4,
  "templates": [
    {
      "name": "Push（推）",
      "sort_order": 0,
      "schedule_rule": { "day_in_cycle": 1 },
      "exercises": [

## 6 业务规则

### 6.1 日历推算逻辑

**weekly 模式**：
- 输入：from_date, to_date, day_of_week 列表
- 遍历日期范围，匹配 weekday() ∈ day_of_week 的日期
- 排除已手动取消的日期

**cyclic 模式**：
- 输入：from_date, to_date, cycle_length, plan_created_date
- 计算 offset = (date - plan_created_date).days % cycle_length
- 匹配 offset == template.schedule_rule.day_in_cycle 的日期

**flexible 模式**：
- 不自动生成条目
- 仅从 plan_schedule_entries 读取手动安排

### 6.2 按模版训练流程

模版作为**动作过滤器**，而非自动生成预填数据的来源。不提供"从模版导入预设组数/重量"功能。

#### 入口 ① — 今日计划 / 日历条目（带 template_id URL 参数）

| 步骤 | 前端 | 后端 |
|------|------|------|
| 1 | 首页「开始」/ 日历「开始」→ `navigate('/workouts/new?template_id=X')` | — |
| 2 | WorkoutCreatePage 读取 URL 参数，调用 `GET /plans/templates/{X}` 加载模版 | 返回 TemplateOut（含 exercises 列表） |
| 3 | 动作列表 `.filter(e => templateExerciseIds.has(e.id))`，**仅显示**模版内动作；非模版动作**不渲染** | — |
| 4 | training/rest 阶段：显示该动作的 `note` 字段内容 | — |
| 5 | 训练完成，`POST /workouts`，payload 含 `plan_template_id: X` | 写入 workouts.plan_template_id；WorkoutSummary 返回 template_name |

#### 入口 ② — 开始训练主入口（无 URL 参数，页内选择）

| 步骤 | 前端 | 后端 |
|------|------|------|
| 1 | 首页「开始训练」→ `navigate('/workouts/new')` | — |
| 2 | WorkoutCreatePage 首屏展示计划/模版选择器（可选）：调用 `GET /plans` 展示激活计划 | 返回 PlanSummary[] |
| 3 | 用户展开计划 → 懒加载 `GET /plans/{plan_id}` → 展开模版列表 | 返回 PlanOut（含 templates） |
| 4 | 用户选择模版 → 激活过滤（同入口①步骤 3-5） | — |
| 5 | 用户可跳过选择，直接从**全动作库**选择（无模版过滤） | — |

#### 过滤行为规范（两种入口通用）

- **正确实现**：`.filter(e => templateExerciseIds.has(e.id))`，未包含在模版中的动作**不出现**在列表中
- **禁止实现**：将模版动作 `.sort()` 排序到前面，保留全部动作可见
- 用户可在已过滤列表内进一步按分类筛选或搜索
- 用户点击「清除选择」→ 回到全动作库（无过滤）

### 6.3 计划完成率计算

```
完成率 = 已完成条目数 / 应完成条目数 × 100%
```

按周/月统计，展示在计划详情页。
